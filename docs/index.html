Operations

Pub /

Accepts one of the following messages:

#1 Subcribe subscribe

Subscribe to a Performance ID

Payload
object
action
string

Indicates the payload is for subscribing to a performance

Const: "subscribe"
performance_id
string
must match ^[ABCDEFGHJKLMNPQRSTXYZ23456789]{6}

A 6 character letter & number string unique to a performance

Additional properties are allowed.

#2 Register Performance register

Register a new performance. Will cause a return of a token and performance ID to be used when publishing

Payload
object
action
string

Inidcates the payload is for a register operation

Const: "register"
artist
string

The Artist's name- can be anything

title
string

The title of the performance- can be anything

performance_date
integer

The starting time of the performance, represented by a Unix timestamp

Additional properties are allowed.

#3 Update Performance update

Update a performance token with new information. Performance ID will stay the same. Server will respond with a register_return message with the new token

Payload
object
action
string

Indicates the payload is for a token update operation

Const: "update"
token
string

A JWT token used to authorize publishing of heartbeat data to a performance.

artist
string

The Artist's name- can be anything

title
string

The title of the performance- can be anything

performance_date
integer

The starting time of the performance, represented by a Unix timestamp

Additional properties are allowed.

#4 Publish Heart Rate publish

Publish a heartrate to all of the subscribers. Must have a valid token (from calling 'register') in order to publish. Performance ID is encoded in the token. TODO: Enumerate error conditions related to expired tokens and all that stuff...

Payload
object
action
string

Inidcates the payload is for a publish operation

Const: "publish"
token
string

A JWT token used to authorize publishing of heartbeat data to a performance.

heartrate
integer
>= 20 <= 230

A heartrate

Additional properties are allowed.

Sub /

Accepts one of the following messages:

#1 Register Return Values register_return

Sent after registering a performance. Contains a token and performance id to allow publishing

Payload
object
action
string

Indicates this payload is a response to registering or updating a performance

Const: "register_return"
token
string

A JWT token used to authorize publishing of heartbeat data to a performance.

performance_id
string
must match ^[ABCDEFGHJKLMNPQRSTXYZ23456789]{6}

A 6 character letter & number string unique to a performance

Additional properties are allowed.

#2 Heartrate Update heartrate_update

Sent to all subscribers when a new heartrate is published

Payload
object
action
string

Indicates the payload is for new heartrate information

Const: "heartrate_update"
heartrate
integer
>= 20 <= 230

A heartrate

Additional properties are allowed.

#3 Subscriber Count Update subscriber_count_update

Broadcast to all of the subscribers of a performance whenever the total number subscribers changes

Payload
object
action
string

Indicates the payload is for updating the number of subscribers to a performance

Const: "subscriber_count_update"
performance_id
string
must match ^[ABCDEFGHJKLMNPQRSTXYZ23456789]{6}

A 6 character letter & number string unique to a performance

active_subscriptions
integer
>= 0

The number of active subscriptions for the performance

Additional properties are allowed.
